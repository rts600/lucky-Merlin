# Lucky 路由器专属构建对比报告

由于本地环境未安装 Go 工具链，最终的二进制打包需要依靠已经配置好的 GitHub Actions。本报告详尽对比了原版构建与当前为华硕（官改/梅林）系统量身打造的极度优化构建差异。

## 模块与功能对比概览

| 原始默认构建模块 | 当前优化构建模块 | 模块名称及功能作用 | 优化或保留说明 | 其他说明 |
|---|---|---|---|---|
| **Reverse Proxy** | **保留** | **反向代理模块**：实现内外网穿透、端口映射到 80/443 | 路由核心功能，必须保留。 | |
| **Port Forward** | **保留** | **端口转发/Socket 代理**：包含 STUN 穿透、TCP/UDP 代理 | 无公网 IP 路由神器，极度依赖，必须保留。 | |
| **DDNS** | **保留** | **动态域名解析**：自动同步公网 IP 到阿里云、腾讯云等域名解析商 | 路由核心功能，必须保留。 | |
| **SSL Certificate** | **保留** | **证书管理**：自动申请和续期 Let's Encrypt / 阿里云等 HTTPS 证书 | 用于 Web 访问的安全防护，非常实用，必须保留。 | |
| **WOL (全量版)** | **已精简** | **网络唤醒**：通过网络发送魔术包唤醒睡眠的局域网设备 | 移除了庞大且路由端完全用不上的第三方云端唤醒（见下方详细）。 | |
| **Blinker (点灯科技)** | **已彻底移除** | **IoT 云端协同**：通过第三方云平台的 MQTT 服务下发指令实现唤醒 | 极大拖累体积与内存。路由用户通常通过内置反代管理面板，因此彻底剔除。 | 移除后释放大量资源。 |
| **Bemfa (巴法云)** | **已彻底移除** | **IoT 云端协同**：同样基于 MQTT 的第三方控制端。 | 同上，彻底剔除。 | 移除后释放大量资源。 |
| `paho.mqtt.golang` | **已彻底移除** | **MQTT 底层通信库**：为上述 IoT 服务提供底层协议支持 | **重型依赖**：由于上层调用被干掉，底层依赖一并被砍。 | 极其消耗 CGO 和运行时内存，移除后内存断崖式下降。 |
| `gopsutil` | **已替换为原生库** | **系统状态拉取库**：由于要兼容跨系统收集 CPU/RAM，含有大量 CGO 桥接 | **重型依赖**：在原版中仅用于后台仪表盘展示 CPU 内存。已替换为轻量级的 `runtime.MemStats` 原生实现。 | 解决跨编译平台臃肿问题，极大缩减二进制体积。 |
| **嵌入式 Web UI** | **普通版保留 / Core 版移除** | **后台管理面板**：内嵌到二进制文件中的 Vue/React 前端页面 | 根据您的需求，通过 Go Build Tags 实现了**双重编译**：标准版包含本地 UI，Core 版彻底移除 UI 变为纯接口端。 | Core 模式体积最小，打开管理页面自动调用云端公网前端；普通模式则适合断网本地调试。 |

## 具体修改的文件及原理说明

为了保证您后续再次打包时不出问题，这里记录了所有的修改细节：

1. **`module/wol/conf/device.go` 等 WOL 核心文件**
   - **修改原因**：彻底剔除 Blinker 和 Bemfa 支持。
   - **修改方式**：删除了 `WOLDevice` 结构体中的相关参数（如 `IOT_DianDeng_Enable`、`IOT_Bemfa_Enable` 等），以及配套的 `BemfaClientStart/Stop` 等方法。同步清除了这些无用字段的初始化赋值。
   
2. **`module/wol/httpapi/api.go`**
   - **修改原因**：剔除 API 层面对外暴露的 IoT 状态。
   - **修改方式**：在 `DeviceInfo` 返回对象中，去掉了对客户端状态的返回，并且去除了相关秘钥的非空校验（因为前端即便传递也已失效）。

3. **`web/web.go` 及拆分出 `web_ui.go` 与 `web_core.go`**
   - **修改原因**：第一为了剥离臃肿的 `gopsutil` 库，第二为了灵活编译是否包含本地界面的“双版本”。
   - **修改方式**：
     - 去除头部的 `github.com/shirou/gopsutil/v3/` 相关的导入。
     - 在 `status()` 方法中，将获取当前机器内存和 CPU 的复杂代码替换为几行极简的 `runtime.ReadMemStats(&m)` 原生调用。
     - **构建标签隔离（核心亮点）**：将原来的 `//go:embed adminviews/dist` 代码分离出来。写到了 `web_ui.go` 中，并加上 `//go:build !core`。同时创建了 `web_core.go` 加上 `//go:build core`。当编译 `Core` 版时，只会将根路径重定向到官方统一线上前端 `http://lucky666.cn/admin/`，大幅瘦身。

4. **`go.mod`**
   - **修改原因**：清理不再需要的庞大上游依赖模块。
   - **修改方式**：剔除了 `gopsutil` 与 `paho.mqtt.golang` 引用。

5. **`.github/workflows/build-asus-merlin.yml`**
   - **修改原因**：实现跨平台的双轨自动打包。
   - **修改方式**：添加了 Github Action 脚本。矩阵涵盖 `arm64`, `armv7`, `armv5` 架构。在内部通过两次编译（普通指令及附带 `-tags core` 指令），分别生成普通版和精简版，随后运行 `upx --best --lzma` 极限压缩并生成 Release。
